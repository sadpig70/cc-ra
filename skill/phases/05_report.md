# Phase 05 — Report (v0.2)

> Triage 산출을 사람이 읽을 수 있는 markdown 보고서로 변환.
> 5분 안에 핵심 파악, 30분 안에 모든 finding 정독 가능한 구조.
>
> **v0.2 핵심 변경**:
> - finding 좌표가 `feature_path::hazard_point` 형태로 표기
> - 각 finding 에 concurrence_count + observing_positions 표시
> - Symbolic trace 파일 링크 포함 (`_traces_<position>/<id>.pg.md`)
> - Phase 3.5 design drift 섹션 별도 (카테고리 H 전용)

---

## 입력

- `{WORKSPACE}/.cc-ra/_findings.json`
- `{WORKSPACE}/.cc-ra/_context.json`

## 산출

- `{WORKSPACE}/.cc-ra/REPORT-{ISO_DATE}.md` — 메인 보고서
- `{WORKSPACE}/.cc-ra/status.json` — 분석 메타데이터
- `{WORKSPACE}/.cc-ra/findings.db.json` — incremental 비교용 (선택)

1. **Critical (priority ≥ 0.5)** — 즉시 검토 후 수정 결정

3. **Spec drift (H)** — 명세 업데이트 또는 코드 정렬 결정

## Step 5.1 — Render

```bash
python "${CC_RA_LIB}/render_report.py" \
  --findings "${WORKSPACE}/.cc-ra/_findings.json" \
  --context "${WORKSPACE}/.cc-ra/_context.json" \
  --output "${WORKSPACE}/.cc-ra/REPORT-${DATE}.md"
```

또는 너 직접 위 markdown 템플릿을 채워 Write.
